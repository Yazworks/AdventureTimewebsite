Testing

Text

Tested Feature Issue Found Change Made
Model file paths Some models did not appear because the file paths were incomplete, misspelled, or did not match the actual folder structure. I changed the model loading system to use full paths such as assets/models/BMOwave.glb and checked the exact spelling of each file.
Animation playback Animations were playing automatically when the model page first loaded. I updated the loadModel function with an autoPlay option so animations only trigger when the user clicks a button.
Duplicate canvas issue The page displayed a strange duplicated background/scene because there were two canvas elements with the same ID. I removed the duplicate canvas and kept a single threeCanvas element for the Three.js renderer.
Lighting GUI The screen went black when the dat.GUI lighting panel was added because the GUI container was missing or the renderer setup was in the wrong order. I added the gui-container element, checked the script order, and corrected the renderer setup so the lighting GUI worked with the Three.js scene.
Peppermint Butler spell effect The first purple mist effect looked like small dots rather than smoke. I replaced the particle effect with larger transparent Three.js sprite clouds to make the spell effect look more like magical smoke.
Enchiridion gem effect The gem glow originally appeared too early and used one generic yellow colour. I delayed the glow effect and assigned individual emissive colours to the gem materials so they glow in sequence.
Audio playback One of the sound effects was not playing correctly because I had accidentally created duplicate JavaScript functions, meaning one version overrode the other and the sound call was lost. I removed the duplicate function and kept one final version that loads the model, triggers the animation, plays the correct MP3 file, and then runs any extra effects.
Wireframe toggle The wireframe mode caused some materials and faces to display incorrectly because the first version assumed each mesh only had one material. I updated the wireframe function so it checks whether a mesh has a single material or an array of materials before applying wireframe mode.
JSON content loading The About page placeholders were not updating because some text was still being pulled from hardcoded JavaScript values instead of the JSON file. I added the correct element IDs in the HTML and updated the JavaScript so the About page text and model descriptions could be loaded from JSON properly.
